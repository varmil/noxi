# 自動テスト実行

## 作業完了後の必須ステップ

すべてのコード変更作業と Lint 修正が完了した後、以下のステップを**必ず**実行してください：

### 1. web ディレクトリでのテスト実行（実装されている場合）

```bash
cd web && npm test
```

### 2. backend ディレクトリでのテスト実行（実装されている場合）

```bash
cd backend && npm test
```

### 3. テストエラーがある場合の対応

- テストが失敗した場合は、エラーメッセージを確認して修正
- 修正後、再度該当ディレクトリで`npm test`を実行
- すべてのテストがパスすることを確認

### 4. 完了確認

- web ディレクトリのテストが全てパス
- backend ディレクトリのテスト（存在する場合）が全てパス
- 「All tests passed」または類似のメッセージが表示されることを確認

## 実行タイミング

- Lint 修正完了後
- Git コミット前
- 作業の最終段階

## 注意事項

- このステップは Lint 修正後、Git コミット前に実行する
- テストが失敗している状態で作業完了とは見なさない
- テストエラーが解消できない場合は、その旨をユーザーに報告し、解決方法を提案する
- 各ディレクトリに移動してからテストを実行する（ルート階層では実行しない）

## 自動化の流れ

1. 作業完了
2. Lint 修正実行
3. **web ディレクトリでテスト実行（該当する場合）**
4. **backend ディレクトリでテスト実行（該当する場合）**
5. Git コミット実行
6. ユーザーに完了報告
