# 自動 Git コミット

## 作業完了後の必須ステップ

すべてのコード変更作業と Lint 修正が完了した後、以下のステップを**必ず**実行してください：

### 1. 変更ファイルの確認

```bash
git status
```

### 2. ステージングエリアに追加

```bash
git add .
```

### 3. 適切なコミットメッセージでコミット

作業内容に応じて以下のフォーマットでコミットメッセージを作成：

#### コミットメッセージの形式

```
<type>: <description>

[optional body]
```

#### タイプの種類

- `feat`: 新機能の追加
- `fix`: バグ修正
- `refactor`: リファクタリング（機能変更なし）
- `test`: テストの追加・修正
- `docs`: ドキュメントの更新
- `style`: コードスタイルの修正（フォーマット、セミコロンなど）
- `chore`: ビルドプロセスやツールの変更

#### 例

```bash
# 新機能追加の場合
git commit -m "feat: ユーザー認証機能を追加"

# バグ修正の場合
git commit -m "fix: フォームバリデーションエラーを修正"

# リファクタリングの場合
git commit -m "refactor: コンポーネント構造を改善し整理"

# テスト追加の場合
git commit -m "test: フォームバリデーションユーティリティのユニットテストを追加"
```

### 4. コミット実行

適切なメッセージを作成してコミットを実行：

```bash
git commit -m "<適切なコミットメッセージ>"
```

## 実行タイミング

- Lint 修正完了後
- 作業の最終段階
- ユーザーに作業完了を報告する直前

## 注意事項

- コミットメッセージは作業内容を正確に反映させる
- **メッセージ本文は日本語で記述する**（prefix は英語のまま）
- 複数の異なる種類の変更がある場合は、適切に分割してコミットする
- コミット前に必ず変更内容を確認する
- 機密情報や不要なファイルがコミットされないよう注意する

## 自動化の流れ

1. 作業完了
2. Lint 修正実行
3. git status で変更確認
4. git add . でステージング
5. 適切なコミットメッセージでコミット
6. ユーザーに完了報告
